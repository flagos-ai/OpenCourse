# 课程融入指南

本文档帮助教师将 FlagOS 课程内容融入自己的教学中。

## 融入方式

### 方式一：选取部分模块

从入门课程和 5 大模块中选择适合的内容融入现有课程：

| 课程/模块 | 适合融入的课程 | 融入建议 |
|------|----------------|----------|
| AI 大模型通识课（入门课程） | 人工智能导论、前沿讲座 | 3 学时入门介绍 |
| 模块 1：AI 系统软件基础 | 计算机体系结构、并行计算 | 选择异构计算、Triton 部分 |
| 模块 2：高性能算子工程 | 深度学习、GPU 编程 | 选择 Attention、算子融合部分 |
| 模块 3：AI 编译器原理 | 编译原理、系统软件 | 选择 IR、MLIR 部分 |
| 模块 4：分布式训练 | 分布式系统、高性能计算 | 选择并行策略、通信优化部分 |
| 模块 5：性能评测 | 软件工程、系统优化 | 选择性能分析、自动调优部分 |

### 方式二：完整引入课程

系统性引入 48 学时完整课程：

1. 按照课程大纲设计教学计划
2. 使用完整讲义和作业资源
3. 采用推荐的实验材料
4. 按推荐方式组织考核

## 实验设计建议

### 使用现有实验

直接使用我们提供的实验：

- 查看现有实验列表
- 选择适合的实验内容
- 使用在线实验室开展实验

### 自定义实验

根据课程需要设计自己的实验：

- 我们提供算力支持
- 协助配置实验环境
- 提供技术咨询服务

详见 [自定义实验支持](../04-实验材料包/自定义实验支持.md)。

## 考核方式建议

### 推荐考核方式

| 考核环节 | 占比 | 说明 |
|----------|------|------|
| 平时实验 | 40% | 模块实验，验证核心原理 |
| 课程项目 | 30% | 综合性项目，培养实践能力 |
| 期末考试 | 30% | 理论考核，检验知识掌握 |

### 灵活调整

根据本校要求调整：

- 调整各环节占比
- 增加或减少考核项目
- 结合本校评分标准

## 教学建议

### 理论教学

1. 系统讲解核心概念和原理
2. 结合最新论文和技术文档
3. 鼓励课堂互动和讨论
4. 布置作业巩固知识

### 实验教学

1. 提前准备实验环境
2. 讲解实验要求和目标
3. 提供必要的实验指导
4. 及时解答学生问题

### 项目指导

1. 引导学生选择合适的课题
2. 定期检查项目进展
3. 提供技术指导和建议
4. 组织项目答辩展示
