# 合作模式说明

本文档详细说明 FlagOS 高校课程的合作模式。

## 三种合作模式

### 模式一：课程融入

**适合对象**：已有相关课程，希望丰富教学内容的高校

**合作方式**：
- 教师选取部分模块内容融入现有课程
- 根据需要调整教学内容和难度
- 自行设计考核方式

**我们提供**：
- 课件资源（讲义、作业）
- 在线实验室算力支持
- 技术咨询

**适用场景**：
- 计算机体系结构课程融入异构计算部分
- 深度学习课程融入算子优化部分
- 编译原理课程融入 AI 编译器部分

---

### 模式二：完整引入

**适合对象**：希望新增系统软件课程的高校

**合作方式**：
- 采用完整 48 学时课程体系
- 使用全部教学资源
- 按推荐方式组织考核

**我们提供**：
- 完整教学资源包
- 在线实验室算力支持
- 教师培训（视情况）
- 持续教学支持

**适用场景**：
- 新增「智能计算系统」专业课程
- 作为系统软件方向核心课程
- 研究生专业选修课

---

### 模式三：合作共建

**适合对象**：希望深度合作的高校

**合作方式**：
- 共同开发教学内容
- 联合培养学生
- 开展科研合作

**我们提供**：
- 所有模式二的资源
- 科研合作机会
- 学生实习推荐

**适用场景**：
- 建立长期合作关系
- 联合培养研究生
- 共同申报科研项目

## 模式对比

| 项目 | 模式一 | 模式二 | 模式三 |
|------|--------|--------|--------|
| 课件资源 | 部分使用 | 全部使用 | 共同开发 |
| 实验支持 | 提供 | 提供 | 优先支持 |
| 教师培训 | 可选 | 提供 | 优先安排 |
| 算力配额 | 标准 | 标准 | 增加 |
| 深度合作 | 无 | 可选 | 有 |

## 选择建议

### 选择模式一

如果您：
- 已有成熟的课程体系
- 希望部分融入新内容
- 需要灵活调整教学内容

### 选择模式二

如果您：
- 希望新增系统软件课程
- 需要完整的教学资源
- 愿意采用标准课程设计

### 选择模式三

如果您：
- 希望建立长期合作
- 有联合培养学生需求
- 对科研合作有兴趣

## 申请流程

无论选择哪种模式，都可以通过以下方式联系我们：

1. 通过 GitHub Issues 或开发者社区联系我们
2. 沟通确定合作方案
3. 签订合作协议（约定非营利性、资源使用等条款）
4. 填写 [资源申请表](资源申请表.md) 申请计算资源
5. 开始合作
